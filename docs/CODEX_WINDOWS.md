# Codex on Windows

This repo is often worked on from Codex Desktop on Windows with PowerShell. These notes are here to save future agents from losing time to shell and package-manager quirks.

## Package Manager

- Use pnpm through Corepack: `corepack pnpm <command>`.
- Do not assume bare `pnpm` is on `PATH`; in this workspace it may not be.
- The package manager is pinned in `package.json` with `packageManager`.
- Keep `pnpm-lock.yaml` as the source lockfile.
- Do not recreate `package-lock.json` unless the maintainers explicitly decide to move back to npm.

Common commands:

```powershell
corepack pnpm install
corepack pnpm test
corepack pnpm run lint
corepack pnpm run build
corepack pnpm pack --dry-run
```

## PowerShell Habits

- Prefer PowerShell-native commands when inspecting files:
  - `Get-Content path\to\file.js`
  - `Get-ChildItem -Force`
  - `Select-String -Path file -Pattern "text"`
- Prefer `rg` for fast repo search when available:
  - `rg "pattern"`
  - `rg --files`
- Avoid Unix-only assumptions such as `cat`, `sed`, `grep`, or heredocs unless you already know they work in the current shell.
- Paths with spaces, such as `C:\Users\PC User\...`, should be passed as tool arguments or quoted carefully.

## Codex Permissions

- The workspace is writable, but commands that need network or elevated filesystem access may require approval.
- If a needed command fails due to network or sandboxing, rerun it with an escalation request rather than working around the approval model.
- Avoid destructive commands unless explicitly requested. In particular, do not use `git reset --hard` or broad delete commands to clean up.

## Git Workflow

- Check state first: `git status --short --branch`.
- Commit focused changes as work lands.
- Push when the maintainer asks to keep GitHub current.
- Do not bump the npm version, tag, or publish unless explicitly requested.

## Build Output

- `dist/` is generated by `corepack pnpm run build`.
- `dist/` is ignored in git but included in the npm package via `package.json#files`.
- Use `corepack pnpm pack --dry-run` to verify package contents before release work.

## Line Endings

Git may warn that LF will be replaced by CRLF on Windows. That warning is expected in this workspace unless the repo later adds a stricter `.gitattributes`.
